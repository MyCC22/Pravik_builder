# Pravik Builder - Design Spec

## Overview
A chat-first website builder powered by v0's APIs. Users enter their phone number, describe what they want via text or voice, and see a live preview generated by v0. Mobile-first, designed to extend with Twilio for phone-call-driven website building.

## User Flow
1. **Login** - Enter phone number, no password. Lookup or create user.
2. **Project List** - View existing projects or create new one.
3. **Builder** - Core screen. 70/30 split on mobile (preview top, chat bottom). Send text or voice prompts to v0, see live iframe preview.
4. **Deploy** - One-tap deployment via v0 API.

## Tech Stack
- Next.js 14 (App Router) + Tailwind CSS
- Supabase (Postgres) for data storage
- v0 Platform SDK (`v0-sdk`) for chat/preview/deploy
- v0 Model API for supplementary AI (quick edits, prompt refinement)
- OpenAI Whisper API for speech-to-text
- Browser MediaRecorder API for audio capture

## Architecture

### Layered, Modular Structure
```
src/
  features/
    auth/           -> login, session, phone lookup
    builder/        -> chat UI, prompt input, voice recorder, preview iframe
    projects/       -> project list, cards, CRUD
    deploy/         -> deployment trigger, status
  services/
    v0/             -> V0PlatformClient + V0ModelClient
    supabase/       -> typed Supabase client wrapper
    whisper/        -> WhisperClient (audio transcription)
    twilio/         -> TwilioClient (webhook verify, SMS, calls) [future-ready]
  components/
    ui/             -> Button, Input, Card, Loading, etc.
    layout/         -> MobileLayout, DesktopLayout, responsive shell
```

### API Routes (thin handlers -> delegate to services)
- `POST /api/auth/login` - phone number lookup/create
- `POST /api/v0/chat` - create v0 chat, store in Supabase
- `POST /api/v0/message` - send message to v0 chat
- `POST /api/voice/transcribe` - audio blob -> Whisper -> text
- `POST /api/v0/deploy` - trigger v0 deployment
- `GET /api/projects` - list user projects
- `POST /api/webhooks/twilio` - Twilio incoming call/SMS handler [placeholder]

### Supabase Schema
- `users` - id, phone_number, created_at
- `projects` - id, user_id, name, v0_chat_id, v0_project_id, preview_url, created_at, updated_at
- `messages` - id, project_id, role (user/assistant), content, created_at
- `sessions` - id, user_id, phone_number, session_token, source (web/twilio), expires_at, created_at

### Key Decisions
- All API keys server-side only (Next.js API routes)
- v0 `chat.demo` URL as iframe preview (no custom rendering)
- Voice: MediaRecorder -> blob -> Whisper API -> text in prompt input
- Services are stateless, independently testable, swappable
- Twilio webhook route exists as placeholder, ready for Step 2

## Mobile Layout (Builder Screen)
- Top 70%: v0 iframe preview (full width, interactive)
- Bottom 30%: Chat area with scrollable history + pinned prompt bar
- Prompt bar: text input + mic button + send button
- Mic tap: record -> stop -> transcribe -> populate input

## Desktop Layout (Builder Screen)
- Center: large v0 iframe preview
- Bottom bar: full-width prompt input with mic + send
- Chat history: collapsible side panel

## Twilio Webhook Flow (Step 2)
1. User calls Twilio number
2. `POST /api/webhooks/twilio` receives caller phone number
3. Lookup/create user via auth service
4. Generate session URL: `app.com/build?session=xyz&phone=+1234567890`
5. Send SMS with link via Twilio
6. User opens link -> auto-login -> builder screen
